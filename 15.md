### **Question 1: Why did my original code fail?**

**Your Code:**

```python
def count_substring(string, sub_string):
    result = count(string, sub_string)

if __name__ == '__main__':
```

**Explanation:**
Your code had two main issues:

1. **Infinite Recursion / Naming Conflict:** Inside your function, you called `count(string, sub_string)`. But in your main block, you assigned the result to a variable named `count`. Python got confused because `count` is not a built-in function; you accidentally turned `count` into a variable name, which broke the function call.

> **Note on `.count()`:** While fixing this syntax gets the code to run, Python's built-in `string.count()` only looks for **non-overlapping** matches. If your string is `"ABCDCDC"` and you search for `"CDC"`, it will only return `1` because it consumes the characters of the first match and misses the second one.

### **Question 2: In `string[i : i + sub_len]`, why do we need `i + sub_len`? Why isn't `string[i : sub_len]` enough?**

**Explanation:**
Python string slicing requires an **absolute ending index**, not a length. The syntax is always `string[Start Index : Stop Index]`.

If you use `string[i : sub_len]`, the stopping point stays permanently fixed at the length of your substring. As your loop moves forward and `i` increases, your "magnifying glass" gets smaller and smaller until it completely disappears.

For example, if `sub_len = 3` (searching for `"CDC"`):

* When `i = 0`: `string[0 : 3]` extracts 3 characters.
* When `i = 1`: `string[1 : 3]` extracts only 2 characters (`"BC"`).
* When `i = 2`: `string[2 : 3]` extracts only 1 character (`"C"`).

By using `i + sub_len`, you shift the stopping point forward by the exact same amount you shifted the starting point. This guarantees that your slicing window remains exactly `sub_len` characters wide at every single step.

### **Question 3: For the loop, will `for i in range(len(string) - sub_len)` work instead of adding `+ 1`?**

**Explanation:**
No, it will not work perfectly. It will cause your code to miss a valid match if it occurs at the very end of the string.

This is due to Python's **off-by-one rule** with the `range(start, stop)` function, which counts up to but **does not include** the `stop` number.

Let’s say your `string` is `"ABCDC"` (length 5) and your `sub_string` is `"CDC"` (length 3). The match is at the very end, starting at index `2`.

* **Without `+ 1` (`range(5 - 3)` $\rightarrow$ `range(2)`):** Python only generates the numbers `0` and `1`. The loop terminates before checking index `2`, missing the match entirely.
* **With `+ 1` (`range(5 - 3 + 1)` $\rightarrow$ `range(3)`):** Python generates `0`, `1`, and `2`. The loop successfully reaches index `2`, checks the slice, and counts the final match.
